# Frontend Architecture & Stack Guidelines

> **Related Documents**: See `docs/frontend-workflow.md` for the step-by-step implementation process and sequence diagrams.

## 1. Core Technology Stack
- **Framework**: Next.js (App Router)
- **UI Library**: React 18+
- **Language**: TypeScript (Strict typing is MANDATORY)
- **Styling**: Tailwind CSS
- **Component Library**: Shadcn UI 
- **Icons**: Lucide React
- **Global State**: Zustand
- **Data Fetching / API Cache**: TanStack Query (React Query) + Axios
- **Data Visualization**: Recharts

## 2. Strict Directory Structure
To maintain control and prevent monolithic, unmaintainable code generated by AI or humans, the frontend strictly adheres to the following separation of concerns:

```text
/frontend
 ├── app/                 # 1. ROUTING & ASSEMBLY (Smart Pages)
 │   ├── tanks/           
 │   │   └── page.tsx     # Glues Custom Hooks and Presentational Components
 │   └── layout.tsx       # Global layouts (Navbar, Sidebar)
 │
 ├── components/          # 2. PRESENTATIONAL (Dumb Components)
 │   ├── ui/              # Generic reusable components (Shadcn UI widgets)
 │   ├── tanks/           # Domain-specific visual components (e.g., TankCard.tsx)
 │   └── charts/          # Visualization components (e.g., FermentationCurve.tsx)
 │
 ├── hooks/               # 3. CONTROLLERS (React Query & Logic)
 │   ├── useTanks.ts      # Data fetching, mutations, and caching logic
 │   └── useAddReading.ts 
 │
 ├── services/            # 4. NETWORK LAYER
 │   ├── api.ts           # Axios base configuration and interceptors
 │   └── fermentation.ts  # Pure API calling functions without React context
 │
 ├── models/ (or types/)  # 5. DOMAIN ENTITIES
 │   └── tank.type.ts     # TypeScript interfaces mirroring Python Pydantic models
 │
 ├── utils/ (or lib/)     # 6. PURE BUSINESS LOGIC
 │   ├── calculations.ts  # Math functions (e.g., potential alcohol) - Fully testable
 │   └── formatters.ts    # Date and number formatting
 │
 └── store/               # 7. GLOBAL STATE
     └── useUserStore.ts  # Zustand stores for session, theme, etc.
```

## 3. Strict Rules for AI Generation
When generating frontend code, the AI Agent MUST obey these rules:
1. **No Monolithic Pages**: Never generate a page that contains data fetching, complex business logic, and UI rendering in a single file. 
2. **Data Fetching belongs in Hooks**: Components inside `components/` MUST NOT import `axios` or `useQuery` directly. They should receive data via `props`. Only `app/` pages or specialized container components can consume `hooks/`.
3. **Types Matter**: NEVER use `any`. Define TypeScript interfaces for everything, especially API responses.
4. **Mobile-First Design**: Design using Tailwind's default (mobile) classes first, then use `md:` and `lg:` prefixes for larger screens, as the primary persona is an Enologist in a cellar using a phone.
5. **Atomic Prompting**: The AI must build small components first, verify them, and assemble them later. DO NOT generate 300+ line files.
